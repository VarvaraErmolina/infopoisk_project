Наш проект реализует поиск по текстовому корпусу с использованием разных методов индексации.

**Структура проекта:**
```
app.py # Запуск сайта
main.py  # CLI-интерфейс, запуск поиска
inverted_index.py  # Основной движок и индексы
matrix_index.py  # Матричная реализация
preprocessing_data.py  # Предобработка текста
woman_ru_9_topic.csv # Корпус
woman_ru_9_topics_preprocessed.csv # Предобработанный корпус, создаётся при первом запуске
templates/
├── base.html
├── index.html
├── search.html
└── results.html
static/
└── style.css
```

**Предобработка**

Тексты проходят:

- приведение к нижнему регистру
- очистку
- лемматизацию
- удаление стоп-слов

Результат сохраняется в колонку preprocessed_text, все индексы строятся по ней.

**Индексы:**

*inverted*

- frequency - частотный индекс
- bm25 - вероятностная модель
- word2vec - усреднение векторов слов
- fasttext - улучшенный word2vec (с n-граммами)
  
*matrix*

- frequency
- bm25

**Параметры:**

- --query - текст запроса
- --engine - inverted / matrix
- --index - тип индекса
- --data - путь к CSV
- --text_column - колонка с текстом
- --top_k - число результатов

пример запуска в CLI:
```python
python main.py \
  --query "гадалка нагадала" \
  --engine inverted \
  --index bm25 \
  --data data/file.csv \
  --text_column preprocessed_text \
  --top_k 5
```

Результат - таблица (doc_id, score, search_time_seconds, текст документа).

Также у проекта есть веб-интерфейс - сайт с:
- главной страницей с описанием и кнопкой-ссылкой на поиск
- страницей поиска с выбором параметров
- страницей с результатами поиска, где есть список текстов, время поиска и оценка близости
