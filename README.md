Наш проект реализует поиск по текстовому корпусу с использованием разных методов индексации.

**Структура проекта:**
```
main.py                  # CLI-интерфейс, точка запуска
preprocessing_data.py    # Предобработка текста
inverted_index.py        # Основной движок и индексы
matrix_index.py          # Матричная реализация
data/*.csv               # Данные
```

**Предобработка**

Тексты проходят:

- приведение к нижнему регистру
- очистку
- лемматизацию
- удаление стоп-слов

Результат сохраняется в колонку preprocessed_text, все индексы строятся по ней.

**Индексы:**

*inverted*

- frequency - частотный индекс
- bm25 - вероятностная модель
- word2vec - усреднение векторов слов
- fasttext - улучшенный word2vec (с n-граммами)
  
*matrix*

- frequency
- bm25

**Параметры:**

- --query - текст запроса
- --engine - inverted / matrix
- --index - тип индекса
- --data - путь к CSV
- --text_column - колонка с текстом
- --top_k - число результатов

пример запуска:
```
python main.py \
  --query "гадалка нагадала" \
  --engine inverted \
  --index bm25 \
  --data data/file.csv \
  --text_column preprocessed_text \
  --top_k 5
```

Результат - таблица (doc_id, score, search_time_seconds, текст документа).
